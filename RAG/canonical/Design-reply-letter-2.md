# Reply letter 2 — rev 0.18/0.19 asks answered, three confirmations, one reversal

Wireframe references (13a–14b) point into the enclosed standalone
wireframes; anchors (`#13a`) work. Turns 13–14 are the library/export
surface; earlier turns unchanged.

## The flagged judgment calls

1. **Text-first link following — confirmed, gladly.** "No UI" meant
   "no *chooser* UI." A link's payload is the note; flying without
   opening would reduce link-following to a camera move. One placement
   ⇒ fly there AND the note opens tethered at the destination; the
   anchor-at-link-then-re-tether mechanic is right. Spatial-only is
   argued against.
2. **Subject-node tag chips — accepted.** Strict superset of my
   arrived-from rule; covers doorless opens (quick-open, tag rows,
   uses rows). Zero-node note ⇒ no chips.
3. **Charm corner — keep it in the settings inventory**, strike from
   the deferred ledger. It was drawn among 10b's sixteen; killing it
   re-litigates a settled sheet for one line's savings.

## Reversal from the owner (supersedes rev 0.19 appendix)

**Nested tags are dropped, not deferred-with-machinery.** Tags stay
flat, globally unique. Ask 6 is void; the include-descendants toggle,
drag-to-nest management surface, and "goblins or all monsters"
glanceability problem all evaporate with it. The gallery keeps a flat
tag list with counts (the Allusion sidebar compressed into a facet
dropdown) if tag counts outgrow memory. If hierarchy returns someday
it is a tags-domain question, not a browser question.

## Asks 1–5 (turn 13)

1. **Gallery** (13a) — the browser is a takeover with a top mode
   switcher: ⊛ graph · ▤ outline · ⊞ gallery are projections of one
   database; enter through any charm, hop freely inside. The gallery
   still gets its own rail charm — it is the migrating-from-Allusion
   user's primary room, not a sub-mode. Facets: sort (date/name/size),
   tag filter, and the two cleanup filters we already own (untagged ·
   unplaced). Bulk select ⇒ floating action bar (tag · place · trash).
   Thousand-file drops run as an interruptible progress strip with a
   live hash-dedupe count — never modal. **Naming: "library" is the
   project kind; "gallery" is the view** — any project can open its
   gallery; "browse your library in the gallery."
2. **Tag border** (13b) — the none/all/pick control is
   **session-scoped in the source panel's header**, set once, applied
   to every pull; never a per-drag interrupt. Defaults: **all** from a
   library, **none** from a world (curation facts travel; world
   context doesn't). "Pick" opens a checklist that persists for the
   session. OS drops have no source tags — no border moment (but see
   mirror moment 2).
3. **Sourcing chrome** (13b) — every row in the project charm's menu
   gets two actions: **switch** and **open as source**. A source opens
   as a **pinned panel** (the compressed gallery: same facets, mini
   grid), obeying panel physics — drag, pin, close. Dragging out
   copies bytes in, hash-deduped.
4. **Seed** (13c) — ordinary records: a root board of public-domain
   artists (Doré, Mucha, Hokusai), each diving to a per-artist board
   with placed, tagged, noted works. The explainer is an ordinary
   pinned note whose one power is the "clear the example" action (an
   ordinary trash command over the example records + itself).
   Unmissable because it's open; not naggy because it's just a note;
   gone forever after clearing. The tutorial is made of the app's own
   furniture — which is itself the lesson.
5. **Q20 closed** (13d) — the pin is a dock tool (◉, shortcut N):
   click places a dot node with its phantom note already focused; type
   to keep, Esc and it never existed. Same teardrop as the bookmark
   button — **pins mean places, everywhere**. The Create Pin dialog
   retires.

## Asks 7–8 (turn 14)

7. **Mirror moments** (14a) — both ride the drop, neither blocks it.
   Moment 1: the once-per-project ask is a two-button panel anchored
   to the **first drop**; import runs either way; the answer is stored
   and changeable via one settings line ("mirror drops to library ✓").
   Moment 2: hash recognized ⇒ a transient chip beside the fresh node
   offers the library's tags ("in your library · #doré #ink · apply /
   ✕"). It obeys the engagement fade: **ignoring it is the dismissal
   gesture** — it dissolves at the next idle, no dismissal debt. Bulk
   drops collapse to one summary chip ("14 known — apply tags?").
8. **Export** (14b) — a sheet anchored to ☰ (not a takeover: you're
   leaving, not browsing). Three sections mirror the three record
   kinds: notes → Markdown (sub-option: append uses section with
   `![[…]]`), files → bytes + tag manifest (sub-option, default OFF:
   tag-folder copies — duplicative by design), boards → full-res
   renders. Size is a **live footer fact on every export**; past the
   threshold the first export adds one acknowledge line, confirmed
   once per project, never a gate (§16).
