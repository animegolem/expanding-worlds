EXPANDING WORLDS · THE NORTH STAR · READ BEFORE YOUR FIRST UI PULL REQUEST

Chrome is a terminal. The world is a desk.

Every pixel in this app is made of one of two materials. The UI around the canvas is a flat, mono-flavored terminal that names itself and never emotes. The canvas holds objects with gentle physics — they lift, settle, tear, press, and burn. Know which one you're working in before you write a line.

LAYER 01

the terminal

rail · dock · menus · tooltips · search · takeover chrome · panels

Shipping a UX-layer feature? You are working in mono. Flat surfaces, unicode glyphs, printed shortcuts, sentence case, verbs. Chrome animates exactly one property — opacity — and everything fades on one shared clock, together, like a tide.

LAYER 02

the desk

art · ink · paper notes · object icons · the lit stage

Shipping a canvas-level feature? You are working in physics. Things the user owns are objects with mass. State changes are physical events you can watch happen — and the hardware left behind tells you what state a thing is in: rings mean bound, tape means on your glass, a push pin means planted in the world.

THE RULES — THIS IS THE PART YOU BUILD FROM
EVERY CANVAS MOUSE-DOWN → ITS BEAT
grab → lift — shadow on, +1% scale, ~120ms
release → settle — flat, one ease-out ~150ms, no bounce
import lands → settle + bloom — stage edge grows ~240ms
snap engage → nudge — magnetic seat, last px ease ~40ms
lock → press — −1% into the desk; grab = 2px strain, no lift
note gestures → tear family — cover ~200 · tear ~300 · tuck ~200 · pin-through ⇄ pull
delete → lift, away — up + fade ~180ms; NEVER a crumple (trash keeps it whole)
pan flick → glide — real friction, ≤300ms, no rubber band
double-click picture → dive — eased flight into the child board
drag over a frame → members make room — the one allowed anticipatory motion
no beat, on purpose: resize · rotate · crop · marquee · draw · pan-grab
THE HAND — CURSOR ZONES, NOT HANDLES
inside moves · edge stretches · corner scales · just outside a corner rotates
Option at drag-START duplicates · Option MID-drag slips the snap
Shift = tidy constraints, and it silences snapping while held
rotation magnetizes to cardinals · Shift quantizes to 15° steps
locked shows the refusal cursor and draws nothing else
THE CLICK GRAMMAR + ESC
single click = select + charm bar · a charm opens exactly its facet
double-click = everything the thing has (dive + note)
esc peels one layer at a time: selection → lens → takeover → nothing
PAPER + ZOOM CONSTANTS
the page opens where the image's shape says: portrait → beside (roomier side) · wide → below, calendar-bound; it matches the SHARED edge, chosen at open, never flips mid-zoom
lifecycle fully reversible — one undo per transition; the torn edge stays
zoom: world shrinks honestly · chrome holds size · ALL furniture on one threshold (EW_FURNITURE_MIN_PX ~8) · page fades whole at EW_PAGE_FLOOR_PX ~48
WHICH MATERIAL AM I IN? — THREE QUESTIONS
1 · does the user OWN it?
They placed it, wrote it, made it → desk. The app provides it as a control or container → terminal.
2 · should the state change be VISIBLE as an event?
The desk's job is making state changes obvious, playfully — bound → torn → pinned. Terminal state changes by color and fill, silently.
3 · would the metaphor ever LIE?
If the physical story misrepresents the system, it's forbidden even on the desk. An honest terminal beats a lying desk, always.
