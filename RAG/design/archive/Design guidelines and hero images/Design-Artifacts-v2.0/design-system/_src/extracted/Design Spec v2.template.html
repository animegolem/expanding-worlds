Expanding Worlds — UI & Interaction Design Spec

v2 · July 2026 · Supersedes v1 and both reply letters; consolidates wireframe turns 1–14. Written to live in the repo beside RFC-0001 (rev 0.19). Where this document and any earlier artifact disagree, this document wins.

Motivation, the sentence that decides our calls: the app is PureRef until you learn it nests; the global views are the user consenting to act against the database; the payoff moment is "wait — THAT'S what I've been building?" The target user is the person who has that moment.

0 · Design stance

  • The window is the board. Chrome is minimal, floating, and never causes canvas reflow. Neutrality is the product: the tool gets out of the way of the art.
  • Mac is the lead platform; Windows/Linux gain a ☰ menu entry point, no other divergence. Glass theme is Mac-only (falls back to dark).
  • One physics rule: panels grow out of the control you pressed — charms, links, menu items, rail icons all open panels anchored to themselves.
  • Nothing has invisible state. Every grouping, link, mode, or ongoing condition is visible on some surface.
  • Opinions ship as defaults, not settings (§9). Numeric thresholds herein are provisional feel numbers.

1 · Shell layout

  • Traffic lights upper-left. Beside them, the navigation path (§2).
  • Mode charm rail, vertical, upper-right: project ⧉ · search ⌕ · graph ⊛ · gallery ⊞ · outline ▤ · menu ☰. Ongoing-condition charm appends below when present (§8).
  • Dock, bottom-center: select · text · shape (press-hold flyout) · draw · line · arrow · connector · pin ◉ · divider · zoom out · % · zoom in · fit. Layer send-forward/back join only while a selection exists.
  • No docked sidebar, no status strip, no tab bar. Transient errors are toasts; ongoing conditions per §8; multi-board work per §2.
  • Title strip (file/view menus, Win/Linux) is hover-revealed at the top edge.

2 · Navigation (RFC §8.1 chrome)

  • Path, upper-left: renders the entry route (a rendered back-stack), not structural ancestry — containment is a graph with legal cycles and no canonical parent. Clicking a crumb goes back to that board with viewport restored.
  • ⌂ Home is its own button at the path's head → root canvas.
  • Back/forward: primarily gestures — trackpad swipe, mouse 4/5, ⌘[ / ⌘]. Hover-revealed ‹ › beside the path; no permanent buttons.
  • Bookmarks: a tiny map-pin button (cap-height circle-dot, generous hit target) at the path's tail. On press it morphs into a teardrop pointing at the menu it opens — the button is its own anchor tail. Menu rows: drag-handle · name · printed shortcut · ✕. Drag order is the ⌘1–n binding. Stale targets grey out with an "in trash" label, never silently vanish. Bottom row: "＋ bookmark this board".
  • Cross-board flights (uses rows, tag rows, pinned-panel origin labels, graph clicks) are navigation events and enter history.
  • Deferred: hold-⌘ switcher HUD; New Window for side-by-side (deferred, not a non-goal).

3 · Chrome visibility (engagement cadence)

  • Engaged (cursor in window): all chrome visible; node charms ~70% opacity — a glanceable census of which nodes have content, never gated on hovering.
  • Idle (cursor out, or ~4s still): the whole chrome layer — dock, rail, path, charms — fades on one shared clock. The board becomes wallpaper.
  • Charm hover: that charm alone lights to 100% + pointer. No node-hover reveal state exists.
  • Node charms hide when the node's rendered screen size drops below a threshold (not zoom %).
  • Transient offers (§7) ride this fade: ignoring them is the dismissal gesture.

4 · Nodes: charms, selection, cursor

4.1 Content-hint charms

  • Two glyphs: page = has a note · frame = has a nested canvas. At most one of each, side-by-side (never stacked), inset ~4px inside the image corner, lower-right (corner is a setting). Subtle scrim chip for legibility; art reads through.
  • The title below the image never collides with charms. Charms are UI, not pixels — excluded from crop/flip previews and exports.
  • The canvas is a node: a screen-fixed corner charm (lower-left) is the current canvas's page charm — ghost + on cursor approach when empty, solid when the note exists.

4.2 Click grammar

TargetGestureResult
imageclickselect + charm bar
page charmclicknote pops in tethered; current canvas unchanged
frame charmclickdive into the nested canvas only
imagedouble-clickeverything: dive AND the canvas's note opens from its corner charm. Note-only ⇒ note; canvas-only ⇒ dive

Charms are the exploded view of double-click; nothing to memorize.

4.3 Selection + charm bar

  • Selection = thin accent outline. No drawn handles, ever.
  • Charm bar beneath the selection: crop · flip H · flip V · divider · make-canvas · note · tags # · lock. # pops the node's tag chips; a chip opens the tag panel (§6).

4.4 Cursor zones

  • Inside ⇒ move · edge ±4px ⇒ directional resize · corner ⇒ diagonal resize · 4–14px outside a corner ⇒ rotate · empty canvas ⇒ pan. ⌥-drag duplicates; locked ⇒ 🚫. The cursor is the affordance; nothing renders on the node.

4.5 The pin tool (closes RFC Q20)

  • Dock tool ◉, shortcut N: click places a dot node with its phantom note open and focused — type to persist, Esc and it never existed. Same teardrop as the bookmark button: pins mean places, everywhere. The Create Pin dialog retires.

5 · Notes and panels

  • Notes open as tethered panels beside their node; a dashed tail marks the attachment. The panel tracks its node as the camera pans; chrome and type stay screen-scale.
  • One tethered panel at a time — opening another replaces it. Pinned panels accumulate (⇱ converts). Tethered is a glance; pinning is a commitment. No auto-unpin, ever.
  • Phantom lifecycle: empty editor persists nothing until the first keystroke; the hint charm turns solid at that moment.
  • Tag chips at the top of a note panel are the subject node's tags; zero-node notes show none.
  • Note panel header always shows ⌖ n places — click unfolds the uses list in-panel: rows grouped by canvas, Unplaced group last, "here" marking the placement being read (RFC §7.4).
  • Wiki-link following is text-first (RFC §7.3): zero placements ⇒ inline notice at the link ("no placements — open note?" → loose-note panel); one ⇒ fly there AND the note opens tethered at the destination (anchors to the clicked link until spatial resolution lands, then re-tethers); many ⇒ location chooser panel anchored to the clicked link.

5.1 "Which node am I reading?" — the escalation rule

  • The indicator escalates with how broken the spatial link is: tethered ⇒ nothing (the tail speaks) · pinned, node on-screen ⇒ accent halo · pinned, node off-screen ⇒ edge chip pointing at it (click flies home) · node on another board ⇒ origin label in the panel header ("⌂ Northern Reach"; click = cross-board flight, enters history).

6 · Tags

  • Node-scoped, app-managed, project-wide, flat. Assigned many-to-many to nodes; never serialized into note text. Bare placed art is taggable — the flagship case. Flat and globally unique: hierarchy is dropped (owner's call). Note-attached tags deferred, not rejected.
  • Tag panel: search with tab-completion; thumbnail rows spanning every board, unplaced nodes included. Per row: ▤ open note · ⌖ fly to placement. Lens ◎ toggle in the header dims the visible canvas to hits (~20% others), survives pan/zoom until dropped.
  • Entry: # on the charm bar · ⌕ (typing # switches to tag mode) · tag chips anywhere. One destination, one row grammar.
  • Multi-tag queries deferred; one tag at a time.

7 · Global views and the library (RFC §14.4)

  • Two scopes, two physics: node-local content floats as panels; project-global views take over the window. Esc or the charm returns; the camera is untouched. Three charms, one house: ⊛ graph · ▤ outline · ⊞ gallery are projections of one database — the takeover has a top mode switcher; enter anywhere, hop freely.
  • Graph ⊛: force layout; wiki-links are the only edges. Placed art renders as the art past a screen-size threshold; pure notes are dots; orphans (node, no note — can't have edges) wear a dashed ring + ⚯. Filters: hide content-less · orphans only · #tag. Click ⇒ fly to placement (or open note if unplaced).
  • Outline ▤: the world as an indented tree, same glyphs; cycles render as ↺ alias rows; unplaced notes gather in a root loose bin. Same filter chips.
  • Gallery ⊞: the browsable collection — virtualized thumbnail grid; facets: sort (date/name/size) · tag filter · untagged · unplaced. Bulk select ⇒ floating action bar (tag · place · trash). Bulk import runs as an interruptible progress strip with live hash-dedupe count, never modal. "Library" names the project kind; "gallery" names the view — any project can open its gallery.
  • Sourcing: project charm menu rows offer switch and open as source. A source opens read-only as a pinned panel (compressed gallery); dragging out copies bytes in, hash-deduped. The none/all/pick tag-border toggle is session-scoped in the source panel header — never per-drag. Defaults: all from a library, none from a world.
  • Seeded first-open: ordinary records — a root board of public-domain artists diving to per-artist boards with placed, tagged, noted works. The explainer is an ordinary pinned note whose one power is "clear the example" (an ordinary trash command over the example + itself). The tutorial is the app's own furniture.
  • Mirror moments: (1) once-per-project ask rides the first drop, anchored to it, import never blocked, answer stored (settings line). (2) Hash-recognized drops get a transient chip offering the library's tags — it obeys the engagement fade (ignoring = dismissing); bulk drops collapse to one summary chip.
  • Export (escape hatch): a sheet anchored to ☰. Three sections mirror the record kinds — notes → Markdown (+ optional uses appendix with ![[…]]) · files → bytes + tag manifest (+ opt-in tag folders) · boards → full-res renders. Size is a live footer fact; past threshold, one acknowledge per project, never a gate.
  • Figma-style frames: rejected (revisit clause stands). Proximity, tags, and nested canvases cover grouping.

8 · Status

  • Transitions ⇒ toasts (enter + resolve). Ongoing conditions ⇒ a ⚠ charm appended below the rail — exists only while the condition holds, pulses once on arrival, click opens a detail panel anchored to it. Multiple conditions: one charm with a count. The status strip retires when this ships.

9 · Theming and settings

  • Themes: dark (default) · light · glass (Mac-only, falls back dark). Grid: lines · dots · none. Optional flat canvas colors (porcelain / putty / slate / ink). Tokens as CSS custom properties; no theme engine.
  • Settings is a translucent takeover (~82% opaque, inset) — appearance changes apply live to the board visible behind it. No apply/save; commit on click; Esc closes.
  • Inventory: Appearance theme · grid · flat color · opacity — Behavior charm corner · chrome-fade delay/never · snap-to-grid · mirror drops to library (per-project) — Window title strip hover/always/never · border · rounded corners — (Win/Linux) menu in rail vs system bar. Exhaustible in one glance, on purpose.
  • Tooltips: every control shows name + shortcut, ~500ms delay, one chip style app-wide. The GUI is the tutorial for the keyboard app.

10 · Deferred ledger

  • multi-tag queries (AND/OR) · note-attached tags · tag hierarchy (dropped hard) · switcher HUD · New Window / side-by-side · gallery keyboard model · OS-drop importer dialogue · grouping frames (rejected-for-now).
