Expanding Worlds — style guide v2

The whole visual system with its engineering callouts. Compiled at RFC rev 0.54 + the 2026-07-06 design pass. Where this contradicts the design-system readme or the v1 artifacts, THIS wins; where it contradicts the RFC, the RFC wins. Canonical token source stays apps/desktop/src/renderer/theme.css — every proposed token below lands there first, then everywhere else.

■ TECH blocks are the build contract: token names, constants, guard implications. Provisional numbers are marked ~.

1 · The doctrine

Chrome is a terminal; the world is a desk. Chrome is flat, mono-flavored, translucent, and self-effacing: unicode glyphs, flat chips, printed shortcuts, one shared fade clock. World content carries gentle materiality: object icons, taped paper notes with torn edges, small one-shot physical beats. The red glossy pin is the one object admitted into chrome adjacency — and it lives ON the paper (panel pin control), never in the rail, dock, or path, which keep the flat teardrop. Bigger elements are flashier and more visual; text stays focused and thin.

■ TECH · the raw-hex guard test stands: every color below becomes an --ew-* token in theme.css BEFORE use. Object-icon gradients, tape, and torn-paper are world-content colors — theme-independent, defined once like the dot palette. Nothing here relaxes the occlusion contract (§8.8) or the engagement clock (§8.2).

2 · Color

Chrome: near-black blue-grey surfaces in translucency tiers (0.82–0.97 alpha by layer), one blue accent (#4a9df0 dark · #2b6cb0 light), desaturated danger/warn. Notes are light paper on every theme. Links: bound blue · unresolved purple · broken red, each with a 50%-alpha underline token. World content (theme-independent): 8 node dots, 6 canvas flats — and now the object materials.

shipped dots · regularization to oklch(.76 .09 h) proposed, unratified (Icon Document 1h)
■ TECH · tokens to ADD to theme.css:
--ew-void-surface: derived at runtime — color-mix(in oklab, [effective board color] ~78%, black). Never a raw hex; tracks arbitrary per-canvas colors free. Grid in the void renders at grid-line alpha × ~0.4. Void is OFF on glass.
--ew-tape-surface: rgba(214,205,182,.6) · --ew-tape-border: rgba(160,150,125,.4) · --ew-paper-torn: #f7f3ea
--ew-obj-*-hi/-lo gradient pairs (world content): blue a8cbe8/6b9cc4 · gold f2d878/d9ab2e · red f0847c/c0392f · pink e8b0d4/c46a9e · orange eec08a/c87d3e · green aad494/6f9e58; stroke = the -lo step darkened ~20%. Gloss highlight: rgba(255,255,255,.4–.45).
Note heading colors (editor): --ew-note-h1 #2b6cb0 · --ew-note-h2 #3b8a7d · --ew-note-h3 #9c7f2e (~provisional; ride theme tokens at build).

3 · Type

Chrome type stays the platform stack, small and quiet: charm 1rem · tool .95 · toast/panel-base .85 · panel chrome .78 · menu rows .75 · chips .7 · micro .62. Mono (ui-monospace, Menlo) for shortcuts, tags, counts, zoom %, tabular-nums. Weight 600 max for chrome. The one carve-out: note text is Maple Mono — kerned, rounded, with a true cursive italic — loud colored headings, org-style folding presentation over the Markdown carrier.

■ TECH · --ew-font-editor: 'Maple Mono', ui-monospace, Menlo, monospace. BUNDLE woff2 400/400-italic/700 with the app (OFL license file alongside); never fetched at runtime — the no-network posture holds. Editor scale (~provisional): body .85rem/1.65 · h1 1.15rem/700 · h2 1rem/700 · h3 .9rem/700, headings colored per §2. Applies to note panels, big editor, note-card appearance excerpts, and gallery text posts; NOWHERE in chrome.

4 · Space, shape, and borders

Compact rem paddings (.15–.75); controls ~30px square. Radii are a meaning tier, not a taste dial: board cards 3 · buttons/menu rows 4–5 · controls 6–7 · charms/menus/toasts 7 · rail charms 8 · panels/dock 9 · action bars 10–12 · chips/facets 999. Borders 1px everywhere, token-tiered. Selection is a 2px accent outline offset 3px — no drawn handles, ever. The bound page keeps a square corner on its binding edge (radius 0 on the seam side, 9px elsewhere).

■ TECH · --ew-node-radius: 3px stands. New geometry facts: bound-page border-radius: 0 9px 9px 0 (mirrored when the page opens left); binder rings ~11px, 2px stroke --ew-text-muted, punched (fill = board color), straddling the seam; frame title row sits ON the frame's top edge (an item label never does — position is the tell). Frame furniture draws only past a rendered-size threshold (share the charm screen-size rule).

5 · Depth

Shadow = screen-space; flat = world content. Panels 0 6px 22px · pinned 0 10px 30px · menus 0 10px 28px · board nodes soft drop only. New rule: floating things cast shadows — anything torn off or riding a drag (mid-tear note, drag ghost, place cursor) picks up the panel shadow for exactly the duration it floats.

■ TECH · --ew-panel-shadow / -pinned / -menu ship in chrome.css. Add --ew-drag-shadow: 0 12px 26px rgba(0,0,0,.5) (~) applied on drag/tear start, removed on settle — one transition, no lingering elevation on world content.

6 · Motion

Two budgets. Chrome: fades and single pulses only, ease-out, 120–240ms, all on one clock. World: small one-shot physical beats — the tear, the bloom, the stage edge easing outward, the (mused) book-cover open. One beat per user act; never ambient, never looping, no bounces.

■ TECH · shipped constants (feel.ts / chrome.css): fade-delay 4000ms · fade 240ms · chrome-rest .92 · hint-charm-rest .7 · tooltip-delay 500ms · toast 6000ms · perch-pulse 700ms (once).
ADD beat constants: EW_BEAT_TEAR_MS ~300 (ease-out, one-shot) · EW_BEAT_BLOOM_MS ~240 (first placement lights the stage) · EW_BEAT_STAGE_EDGE_MS ~180 (edge follows a push, eased) · EW_BEAT_COVER_MS ~200 (book-cover open, musing). Guard idea: any CSS animation with iteration-count != 1 outside the interaction layer fails review.

7 · The z-ladder (§8.8)

One named ladder, in order: world content → canvas affordances → note panels → takeover → chrome / source panel → anchored popovers → modal → notices → tooltip. Modals mount at the root overlay host. Everything anchored clamps and flips through one shared helper. Corner chrome declares reserved bands; flights reserve space for the panel they open. Chrome never occludes what it annotates.

■ TECH · export Z = { world:0, affordance:100, panel:200, takeover:300, chrome:400, popover:500, modal:600, notice:700, tooltip:800 } (rung names normative, numbers illustrative); a guard test fails any literal z-index that is not a rung constant. The centered torn-off note is a MODAL-rung tenant with a scrim; scroll inside it never reaches the board.

8 · Iconography

Chrome glyphs stay unicode, per surface: rail ⧉ ⌕ ⊛ ⊞ ▤ ☰ (+⚠ perch) · dock ▸ T ▭ ✎ ╱ ↗ ⌁ ◉ − % + ⤢ · charm bar ⌗ ⇋ ⇵ | swatch · ▦ · ▤ · # · ⊘. Hint charms remain drawn bordered-divs. The pin/teardrop motif means "place" everywhere — flat in chrome, glossy object on paper. The six node icons are the painted object set (star · pin · flag · heart · bolt · leaf), degrading to the plain dot below ~8px rendered size.

■ TECH · icons render in the WebGL canvas, not the DOM: bake each object glyph to a small texture atlas (SVG → raster at 2–3 sizes) or draw as geometry; the color choice tints dot and icon alike via the same token. Below ~8px screen size the renderer swaps to the plain dot (same rule family as charm visibility). Objects live nowhere in chrome; the switcher popover shows them at chrome size but they remain world assets.

9 · Component rules digest

Notes: the bound page's height exactly matches its image — like a book; width is the free variable; rings on the seam; corner controls identical everywhere (⌖ places · pin · ⤢); tethered never resizes and scales with the world; pin = the tear (tape + torn edge are the pinned state's permanent marks); pinned resizes, and past a threshold becomes the big editor; double-click tears to center over a dimmed board, scroll stays inside.
Tooltips: five legal shapes only — name+shortcut · name-only · disabled-with-reason · state-with-exit · coming-soon. One chip, 500ms, clamp-and-flip.
Search track: ⌕ icon → bar → committed terms crystallize into pills (AND, booru-style) → trailing ✕ clears all → double-click melts pills to text; empty collapses to the icon.
Frames: title + sort chip ON the top edge, zoom-gated; grid · rows · float; fresh frames default sorted; only item drags across the boundary edit membership.
Settings: trailing fold carets on section headings; "this world" chips on project-tier rows; Advanced is one toggle revealing store → AI; git native, other storage = connectors.
Gallery: scope toggle · facets incl. inbox + low quality · bucketed time with header-as-jump · size slider · Space = Quick-Look peek · import strip centered above the action bar.

10 · Copy

Sentence case everywhere. Never the word "file" — the app copies bytes and must never imply it could touch the user's drive. "Item" where a noun is unavoidable; "node" stays a docs word. Verbs over nouns: Replace image… · Swap for… · open as source · pull into this world. Confirmations state impact as fact ("this will affect 35 placements — are you sure?"). Tooltips teach; menus print shortcuts; no emoji, no exclamation marks, no marketing voice.
